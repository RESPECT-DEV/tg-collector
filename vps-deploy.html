Деплой на VPS

Пошаговая инструкция по настройке сервера и запуску телеграм-бота в продакшне.

1 Выбрать VPS и подключиться

Любой провайдер подойдёт: Timeweb, Selectel, Hetzner, DigitalOcean и др. При создании выбрать Ubuntu 22.04. После создания провайдер даст IP и пароль root.

bash 
# Подключиться с локальной машины
ssh root@<IP_сервера>

2 Первичная настройка сервера

⚠
Работать под root небезопасно — создай отдельного пользователя и используй его.
bash 
# Обновить пакеты
apt update && apt upgrade -y

# Создать отдельного пользователя
adduser elena
usermod -aG sudo elena

# Переключиться на нового пользователя
su - elena

3 Установить Node.js

Устанавливаем через nvm — менеджер версий Node.js.

bash 
# Установить nvm
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.7/install.sh | bash

# Перезагрузить окружение
source ~/.bashrc

# Установить Node.js 20
nvm install 20
nvm use 20

# Проверить
node -v
npm -v

4 Nginx + HTTPS

Nginx принимает запросы на порту 443 (HTTPS) и проксирует их на Node.js (порт 3000).

⚠
Сначала нужен домен — прописать A-запись на IP сервера у регистратора. Только после этого certbot сможет получить сертификат.
bash 
# Установить Nginx и certbot
sudo apt install nginx -y
sudo apt install certbot python3-certbot-nginx -y

# Получить SSL-сертификат (Let's Encrypt, бесплатно)
sudo certbot --nginx -d yourdomain.com

# Открыть конфиг Nginx
sudo nano /etc/nginx/sites-available/default

Привести конфиг к такому виду:

nginx 
server {
    listen 443 ssl;
    server_name yourdomain.com;

    # ssl-параметры certbot уже вписал автоматически

    location / {
        proxy_pass http://localhost:3000;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
}
bash 
# Проверить конфиг и перезапустить Nginx
sudo nginx -t
sudo systemctl reload nginx

5 Загрузить проект

▶
Рекомендую через Git — удобнее обновлять в будущем командой git pull.
bash · вариант А — git 
cd ~
git clone https://github.com/твой-репозиторий/tg-collector.git
cd tg-collector
npm install
bash · вариант Б — scp 
# С локальной машины
scp -r /Users/elena/Desktop/projects/tg-collector elena@<IP>:~/

6 Создать .env на сервере

⚠
.env не хранится в git — его нужно создавать вручную на каждом сервере.
bash 
cd ~/tg-collector
cp .env.example .env
nano .env
.env 
BOT_TOKEN=твой_токен
WEBHOOK_URL=https://yourdomain.com
BACKEND_ENDPOINT=https://r.respectrb.ru/api/tg-message/
PORT=3000

7 Запустить через PM2

PM2 — менеджер процессов: следит чтобы бот работал 24/7, перезапускает при падении, стартует после перезагрузки сервера.

bash 
# Установить PM2
npm install -g pm2

# Запустить бота
pm2 start index.js --name tg-collector

# Автозапуск после перезагрузки сервера
pm2 startup
pm2 save
Команда Действие
pm2 status Статус всех процессов
pm2 logs tg-collector Логи в реальном времени
pm2 restart tg-collector Перезапустить бота
pm2 stop tg-collector Остановить бота

→ Итоговая схема

Интернет ↓ HTTPS :443 Nginx ↓ HTTP :3000 Node.js (PM2) ↓ bot.js → handlers.js → POST на бэкенд

↑ Обновление проекта

bash 
cd ~/tg-collector
git pull
npm install              # если менялись зависимости
pm2 restart tg-collector
